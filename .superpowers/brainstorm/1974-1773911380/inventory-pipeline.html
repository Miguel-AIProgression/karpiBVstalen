Sectie 2: Voorraadbeheer — De Pipeline

Drie voorraadniveaus met doorkijk van snijden tot levering

Productiepipeline

Flow: Snijden → Afwerken → Bundelen → Leveren

  ━━━ NIVEAU 1: GESNEDEN ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────┐
  │    cut_batches           │  Snij-opdracht
  │──────────────────────────│
  │ id                       │
  │ quality_id               │──→ welke kwaliteit
  │ color_code_id            │──→ welke kleur
  │ dimension_id             │──→ welke maat
  │ quantity                 │  hoeveel gesneden
  │ cut_date                 │
  │ cut_by                   │  (operator)
  └──────────┬───────────────┘
             │
             ▼
  ━━━ VOORRAAD: ONGAFGEWERKT (raw_stock) ━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────┐
  │    raw_stock             │  Gesneden, nog niet afgewerkt
  │──────────────────────────│
  │ quality_id               │
  │ color_code_id            │
  │ dimension_id             │
  │ quantity                 │  ← automatisch bijgewerkt
  │ location_id              │──→ waar ligt het?
  └──────────┬───────────────┘
             │
             ▼
  ━━━ NIVEAU 2: AFGEWERKT ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────┐
  │   finishing_batches     │  Afwerk-opdracht
  │──────────────────────────│
  │ id                       │
  │ finishing_type_id        │──→ welke afwerking
  │ quantity                 │
  │ started_at / finished_at │
  │ finished_by              │  (operator)
  └──────────┬───────────────┘
             │
             ▼
  ━━━ VOORRAAD: AFGEWERKT (finished_stock) ━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────┐
  │   finished_stock        │  Grijpvoorraad — klaar om te bundelen
  │──────────────────────────│
  │ quality_id               │
  │ color_code_id            │
  │ dimension_id             │
  │ finishing_type_id        │
  │ quantity                 │  ← automatisch bijgewerkt
  │ location_id              │──→ waar ligt het?
  └──────────┬───────────────┘
             │
             ▼
  ━━━ NIVEAU 3: BUNDELS ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ┌──────────────────────────┐
  │   bundle_stock          │  Complete bundels — verkoopklaar
  │──────────────────────────│
  │ bundle_config_id         │──→ welke bundel-samenstelling
  │ quantity                 │
  │ location_id              │──→ waar ligt het?
  └──────────────────────────┘

Bundel-configuratie

Bundels: standaard + maatwerk per klant
  ┌─────────────────────────┐
  │   bundle_configs       │  Bundel-samenstelling (template)
  │─────────────────────────│
  │ id                      │
  │ name                    │  Bijv. "Milano Standaard Set"
  │ client_id               │──→ NULL = standaard, anders klant-specifiek
  │ is_template             │  herbruikbaar als basis?
  └────────┬────────────────┘
           │ 1:N
           ▼
  ┌─────────────────────────┐
  │  bundle_config_items  │  Wat zit er in de bundel?
  │─────────────────────────│
  │ id                      │
  │ bundle_config_id        │──→ bundle_configs
  │ quality_id              │
  │ color_code_id           │  (NULL = alle kleuren van die kwaliteit)
  │ finishing_type_id       │
  │ dimension_id            │
  │ quantity                │  hoeveel van dit staaltje in de bundel
  └─────────────────────────┘

  Voorbeeld: "Milano Standaard Set" bevat:
    - 10x Milano Velour, kleur 001-010, Blindzoom, 20x20
    - 10x Milano Bouclé, kleur 001-010, Overlocking, 20x20

Locatiebeheer

Fysieke locaties: Gangpad → Stelling → Laag
  ┌─────────────────────────┐
  │   locations            │
  │─────────────────────────│
  │ id                      │
  │ warehouse_id            │  (voor later: meerdere locaties)
  │ aisle                   │  gangpad
  │ rack                    │  stelling
  │ level                   │  laag
  │ label                   │  Bijv. "A3-R2-L1"
  └─────────────────────────┘

  Elke voorraadtabel (raw_stock, finished_stock,
  bundle_stock) heeft een location_id
  → "Waar liggen de afgewerkte Milano Velour 001?"
  → "Gangpad A, Stelling 3, Laag 2"

Het verkoop-perspectief: doorkijk door de pipeline

Verkoop vraagt: "Kan ik 50 bundels 'Milano Standaard' leveren?"

Het systeem kijkt automatisch:

  1. Bundel-voorraad: 20 klaar → 30 tekort
  2. Afgewerkte voorraad: genoeg staaltjes voor nog 15 bundels → nog 15 tekort
  3. Gesneden voorraad: genoeg voor nog 10 bundels na afwerking → nog 5 tekort
  4. Conclusie: "20 direct leverbaar, 45 binnen 3 dagen, 5 moeten nog gesneden worden (5 dagen)"
