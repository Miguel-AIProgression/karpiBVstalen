Sectie 4: Technische Architectuur

Stack, projectstructuur en hoe alles samenhangt

Tech Stack

Backend вҖ” Supabase

  • PostgreSQL вҖ” datamodel, views, functies
  • Row Level Security вҖ” rolgebaseerde toegang
  • Realtime вҖ” live voorraad-updates
  • Auth вҖ” gebruikersbeheer
  • Storage вҖ” klantlogo's voor stickers
  • Edge Functions вҖ” alleen voor complexe berekeningen (levertijd-inschatting)

Frontend вҖ” Next.js + React

  • Next.js 14+ вҖ” App Router
  • TypeScript вҖ” type-safety
  • Tailwind CSS вҖ” styling
  • shadcn/ui вҖ” component library
  • Supabase JS Client вҖ” data + realtime
  • Vercel вҖ” hosting (of self-hosted)

Gebruikersrollen (fase 1+2)

Productie

  • Snij-batches registreren
  • Afwerk-batches registreren
  • Bundels samenstellen
  • Locaties beheren
  • Eigen voorraad-dashboard

Verkoop

  • Bundel-voorraad bekijken
  • Levertijd opvragen
  • Pipeline-doorkijk
  • Klantprijzen bekijken
  • Geen productie-acties

Admin / Management

  • Alles van productie + verkoop
  • Klanten & prijzen beheren
  • Bundel-configuraties maken
  • CommerciГ«le regels instellen
  • Overzichts-KPI's

Slimme database-views (voorbeelden)

v_pipeline_status вҖ” per kwaliteit+kleur+afwerking: hoeveel gesneden, afgewerkt, gebundeld

v_bundle_availability вҖ” per bundel-configuratie: hoeveel klaar, hoeveel maakbaar uit voorraad

v_restock_needed вҖ” welke combinaties onder minimum zitten (voor toekomstige alerts)

v_client_catalog вҖ” per klant: welke producten beschikbaar (technische + commerciГ«le regels gecombineerd)
